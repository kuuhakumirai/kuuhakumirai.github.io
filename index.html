Posted on 2025-11-04

需求

于是用到AssetPostprocessor。

解决方案

思路如下：

  1. 使用OnPostprocessAllAssets函数，遍历importedAssets，判定Asset所在路径；

  2. 获取Asset的AddressableAssetEntry，如果存在，则跳过当前循环；

  3. 根据path获取Asset所在文件夹名，检测是否存在同名的AddressableAssetGroup，如果不存在，则根据文件夹名创建分组；

  4. 使用AddressableAssetSettings.CreateOrMoveEntry函数创建Entry，然后移动至对应的分组中，并将Address设置为文件名，等同于Simplify Addressable Names。

CS文件编码批量转换为UTF-8 / 以UTF-8保存

Posted on 2025-10-26

问题

突然发现项目里CS文件的编码是GBK，文件里的中文字符在Inspector上也显示为乱码。

于是用C#写一个工具，编译后丢到Assets目录里，批量转换。

然而从Unity创建CS文件，文件的编码格式仍然是GBK。那么就要修改设置，以UTF-8的编码保存。

解决方案

1. 修改保存选项

编辑 > 高级 > 设置编码 > 保存时使用编码

2. 设置默认编码

根据Visual Studio IDE 文档：

从 Visual Studio 2022 版本 17.13 预览版 1 开始，可以设置用于保存文件的默认编码。

若要设置默认值，请选择 工具 > 选项 > 环境，文档。 接下来，选择 “保存具有特定编码的文件”，然后选择所需的编码作为默认值。

3. 创建后以UTF-8保存

Unity有一个类叫做AssetModificationProcessor，根据官方说明，这个类可以hook到序列化资产和场景的保存里。

这个类有四个主要的函数：OnWillCreateAsset，OnWillDeleteAsset，OnWillMoveAsset，OnWillSaveAssets。

其中，当Unity要创建一个还没有导入的资产时，会调用OnWillCreateAsset函数。

其中，assetName是资产的相对路径。

到这里思路就清晰了：创建CS文件后，用File类读取文件，再以UTF-8格式保存。

但是，如果在这个函数里使用File类，传入assetName，对创建的CS文件进行读取，会提示错误：FileNotFoundException: Could not find file "xxx.cs"

这是因为，这个函数是OnWillCreateAsset。

在Unity里，还有一个类叫做AssetPostprocessor，根据官方说明，这个类允许hook到导入pipeline，在导入资产之前或之后运行脚本。顾名思义，它是“资产的后处理”。

这个类里有一个函数OnPostprocessAllAssets，在导入任意数量的资产后（Asset进度条到达终点时），此函数被调用。

那么可以尝试在OnWillCreateAsset中标记刚刚创建的CS文件，在OnPostprocessAllAssets中读写文件。

升级到Unity6后TexhMeshPro文字变成白块

Posted on 2025-06-12

问题

使用Unity旧版本中创建的项目，将编辑器升级到Unity6后，TextMeshPro文字变成白块。

解决方式

TextMeshPro使用SDF Shader来渲染，新版本Shader文件计算粗体和缩放所使用的向量分量与旧版本不同。

把材质使用的Shader文件中的texcoord1.y改成texcoord0.z即可。

Drags can only be started from MouseDown or MouseDrag events

Posted on 2025-06-01

问题

为编辑器创建鼠标拖拽事件时，如果判定方式是点击鼠标左键：

private void OnMouseDown(MouseDownEvent evt)
{
    if (evt.button == (int)MouseButton.LeftMouse)
    {
        DragAndDrop.PrepareStartDrag();
        DragAndDrop.SetGenericData("BlackboardField", this);
        DragAndDrop.StartDrag("Dragging Blackboard Field.");
        evt.StopPropagation();
    }
}

会出现报错：

Drags can only be started from MouseDown or MouseDrag events.

解决方式

看一下编辑器源码：

// Start a drag operation.
public static void StartDrag(string title)
{
    if (Event.current.type == EventType.MouseDown || Event.current.type == EventType.MouseDrag)
    {
        StartDrag_Internal(title);
    }
    else
    {
        Debug.LogError("Drags can only be started from MouseDown or MouseDrag events");
    }
}

谜底就在谜面上，把判定的内容从“点击鼠标左键”改成“按下鼠标事件”即可。

把：

if (evt.button == (int)MouseButton.LeftMouse)

替换为：

if (Event.current.type == EventType.MouseDrag)
